# Community Program

## Purpose

The community program defines how external contributors can help improve the
benchmark without weakening hidden tests, evaluator independence, or public
trust.

## Contribution Areas

- documentation improvements
- public fixtures
- harness adapters
- analysis notebooks
- accessibility improvements
- report templates
- bug reports
- public roadmap proposals
- evaluator onboarding interest
- independent replication reports

## Contribution Rules

Contributors cannot access hidden tests by default. Fixture contributions need
review for overfitting risk. Evaluator candidates complete onboarding and
conflict disclosure before scoring official runs.

## Community Channels

Maintain public issue labels, discussion space, roadmap links, contribution
guide, code of conduct, security contact, and recognition page. Sensitive
reports use private security or maintainer channels.

## Recognition

Recognize contributions without implying endorsement of a participant's model,
vendor, or leaderboard result.

## Research Basis

Open Source Guides motivate clear contribution processes and welcoming
community norms. W3C community process norms motivate archived contributions,
public discussion, and transparent participation rules.
