# FAQ

## Purpose

The FAQ answers common public questions in short, durable language. It should
reduce misunderstanding before readers interpret scores or submit results.

## Required Questions

- What is Vibe Browser Benchmark?
- Why benchmark agents by building a browser?
- What do M0-M9 mean?
- What is a blended score?
- Are results comparable across seasons?
- Do hidden tests exist?
- Can agents call a public checker?
- Why are some runs official-unranked or provisional?
- How do I submit a run?
- Can vendors submit their own results?
- What artifacts are public?
- How are secrets and hidden tests protected?
- What does the benchmark not measure?
- How do appeals and corrections work?
- How do I cite a result or data export?

## Answer Style

Answers should be direct, no more than a few short paragraphs, and link to the
authoritative spec for detail. Avoid promotional language and avoid repeating
full methodology text.

## Maintenance

Every FAQ entry has an owner, reviewed date, and source spec. When a source
spec changes materially, related FAQ entries enter review state.

## Public Placement

The FAQ is linked from homepage, methodology, submission guide, leaderboard,
run result pages, and footer. Result-specific questions should deep-link to the
relevant run page when possible.

## Research Basis

NIST AI RMF communication principles motivate clear documentation of intended
use, limitations, and uncertainty. WCAG guidance motivates concise headings,
consistent navigation, and accessible link text.
